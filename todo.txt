Task 3: Client-side E2E Encryption (add helper functions to satellite.py) — standalone, client-side (done)
   - Purpose: provide encryption/decryption helper functions that clients/feeders can use for end-to-end encryption.
   - Use AEAD (AES-GCM) from `cryptography` AEAD APIs.
   - Functions to add:
       - `encrypt_object(data_bytes: bytes, key: bytes) -> dict` → returns `{ciphertext: bytes, nonce: bytes, tag: bytes}`.
       - `decrypt_object(ciphertext: bytes, key: bytes, nonce: bytes, tag: bytes) -> bytes` → returns plaintext.
   - Implementation details:
       - Key validation: must be exactly 32 bytes (256-bit AES-256). Generate with `os.urandom(32)`.
       - Nonce: 12-byte (96-bit) random value generated per encryption to ensure unique ciphertexts.
       - Authentication tag: 16-byte (128-bit) GCM tag automatically verifies integrity and detects tampering.
       - `decrypt_object()` accepts both dict form (returned by encrypt) or separate (ciphertext, nonce, tag) bytes.
       - Raises `cryptography.hazmat.primitives.ciphers.aead.InvalidTag` if ciphertext modified or wrong key used.
   - Flow: client encrypts file → produces ciphertext, store AEAD nonce and key-encryption metadata (key must be kept safe by client operator).
   - Fragmentation: encrypt whole object first, then fragment ciphertext; no fragment alone leaks plaintext.
   - Metadata: store `encryption: { alg: 'AES-GCM', key_id: '<id>', nonce: base64 }` alongside object manifest.
   - Config: none (client-driven)
   - Acceptance: decrypt after reconstruct_file; integration test with end-to-end encrypt→fragment→reconstruct→decrypt.
   ✅ Deliverable: E2E encryption helpers complete.

     - Raw throughput: 6.25 MB/s = 22.5 GB/hour = 540 GB/day

     - Storj total: ~4.5 exabytes across ~12,000 nodes (~375 TB average/node)

     - LibreMesh positioning: much smaller scale, bootstrap/proof-of-concept
     
     OTHER DISTRIBUTED STORAGE COMPARISONS:
     - Sia: ~2-3 PB total, $1-2/TB/month, similar home node model
     - Filecoin: enterprise/datacenter focus, higher costs

     - Reality: bootstrapping with <10 nodes is experimental, not production-ready

     - One big node (8-12 TB):
         + Simpler management, single purchase
